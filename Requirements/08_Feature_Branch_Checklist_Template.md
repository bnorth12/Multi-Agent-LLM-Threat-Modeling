## A. Planning and Traceability

- [ ] Branch is linked to at least one planning issue.
- [ ] Scope in branch matches planning issue acceptance criteria.
- [ ] Requirement IDs implemented by this branch are listed.

## B. Implementation Quality

- [ ] Code changes are reviewed.
- [ ] Unit tests are added or updated.
- [ ] Contract and integration tests pass.
- [ ] Documentation updates are completed.

## C. Requirement Verification

- [ ] Implemented requirements have verification evidence attached.
- [ ] Verification methods used are recorded.
- [ ] Verification statements are satisfied.

## D. Governance and Security

- [ ] Audit-relevant changes are logged.
- [ ] Role and permission impacts are reviewed.
- [ ] No critical unresolved defects remain.

## E. Release Readiness

- [ ] PR references related issue IDs.
- [ ] Related issue status is updated for merge readiness.
- [ ] Branch is rebased or merged with target branch policy.
- [ ] Release notes entry is prepared.
- [ ] Markdownlint passes and evidence is attached (output snippet, screenshot, or CI link).
